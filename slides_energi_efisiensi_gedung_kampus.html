Konteks Masalah
02

Energi kampus besar sulit dikelola tanpa visibilitas granular.

Gedung akademik, laboratorium, asrama, kantor, area makan, pencahayaan, HVAC, dan fasilitas umum memiliki karakter konsumsi berbeda. Beban tinggi tidak selalu berarti boros, tetapi tanpa data sulit membedakan kebutuhan operasional dari pemborosan.

Masalah operasional yang ingin dijawab

  • Kapan beban listrik mencapai puncak?
  • Area mana yang konsumsinya paling tinggi?
  • Apakah ada konsumsi besar di luar jam aktif?
  • Bagaimana prioritas efisiensi ditentukan?
Analitik energi sebagai dasar keputusanProblem framing

06

Scope dibuat realistis: analisis data meter, bukan audit fisik perangkat.

Termasuk

  • Konsumsi listrik per meter, zona, atau gedung.
  • Pola waktu: jam, hari, bulan, weekday/weekend.

Alasan & Keterbatasan
09

Dataset dipilih karena paling cocok untuk analisis energi kampus, tetapi perlu strategi reduksi data.

Kampus nyata Multi-gedung Metadata Brick Load forecasting Fault detection

Kekuatan

  • Sesuai konteks gedung/kampus.
  • Data nyata, bukan simulasi.
  • Periode panjang dan granular.
  • Ada metadata untuk segmentasi.

Risiko awal

  • Ukuran data besar.
  • Metadata `.ttl` Brick Schema perlu dipahami.
  • Ada missing value.
  • Konteks Hong Kong tidak identik dengan Indonesia.
Dataset utama tetap HKUST; subset dipakai agar feasibleSelection rationale
Strategi Subset
10

Mulai dari data clean interval 1 jam atau 1 hari, lalu fokus pada 3-5 gedung/zona.

Strategi ini menjaga ukuran data tetap ringan tanpa menghilangkan pola utama yang dibutuhkan untuk analisis waktu, perbandingan area, dan deteksi anomali awal.

Default rekomendasi: gunakan T60 untuk pola harian dan anomali, lalu T1440 untuk ringkasan manajemen.
Feasible analytics: subset, resampling, metadata joinExecution strategy
Pembagian Peran
11

Empat anggota, empat jalur kerja, satu pipeline analitik.

Anggota 1

Data Engineer

Mengunduh dataset, menata folder, memilih subset, memastikan file dapat dibaca.

Anggota 2

Data Preprocessing Lead

Membersihkan data, menangani missing value, resampling, dan membuat fitur waktu.

Anggota 3

Data Analyst / Modeler

Menganalisis pola konsumsi, membandingkan gedung/zona, dan mendeteksi anomali.

Anggota 4

Visualization + Documentation Lead

Membuat dashboard, menyusun insight, merapikan laporan, dan menyiapkan presentasi.

Documentation digabung dengan Visualization karena anggota 4 orangTeam roles
